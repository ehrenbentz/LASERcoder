# LaserTAG macOS Build Instructions

All commands are run from inside `LaserTAG/build_MacOS/`.

```
LaserTAG/
  CodeBase/            Python source files
  build_MacOS/         Build resources and scripts
    libs/              Bundled dylibs (generated by Step 1)
    laser.ico          Application icon
    collect_dylibs.sh  Dylib collection script
    build_LaserTAG_MacOS.sh   Automated build script
    output/            Build artifacts (generated by Step 5)
      LaserTAG.app
      LaserTAGInstaller.dmg
      LaserTAG.build/
      LaserTAG.dist/
  build_Windows/       Windows build resources
```


## Prerequisites

```bash
brew install mpv
pip install nuitka PySide6 python-mpv
```

Xcode Command Line Tools must be installed (`xcode-select --install`).


## Step 1: Collect dylibs from Homebrew

Skip this step if `libs/` already exists and is populated.

```bash
chmod +x ./collect_dylibs.sh
./collect_dylibs.sh /opt/homebrew/lib/libmpv.2.dylib ./libs
```

Verify:

```bash
ls libs/*.dylib | wc -l
```

Expected: ~47 libraries.


## Step 2: Remove problematic libraries

The collector pulls in `libvapoursynth-script.0.dylib` which depends on
a Python interpreter binary. Both must be removed.

```bash
rm -f libs/Python
rm -f libs/libvapoursynth-script.0.dylib
```

Verify:

```bash
ls libs/Python 2>/dev/null && echo "ERROR: still present" || echo "OK"
ls libs/libvapoursynth-script.0.dylib 2>/dev/null && echo "ERROR: still present" || echo "OK"
```


## Step 3: Create vapoursynth stub

libmpv links against vapoursynth at load time and calls
getVSScriptAPI() on startup to check availability. Returning NULL
tells mpv that vapoursynth is not available. LaserTAG does not use
vapoursynth filters.

```bash
echo 'void* getVSScriptAPI(int version) { return 0; }' \
    | cc -shared -o libs/libvapoursynth-script.0.dylib -x c - -arch arm64
install_name_tool -id "@loader_path/libvapoursynth-script.0.dylib" \
    libs/libvapoursynth-script.0.dylib
```

Verify the stub exports the symbol:

```bash
nm -g libs/libvapoursynth-script.0.dylib | grep getVSScriptAPI
```

Expected output contains `_getVSScriptAPI`.


## Step 4: Codesign all dylibs

```bash
for lib in libs/*.dylib; do
    codesign --force --sign - "$lib"
done
```

Verify:

```bash
codesign --verify libs/libmpv.2.dylib && echo "OK" || echo "FAILED"
```


## Step 5: Prepare output directory

Create a clean output directory and stage the source code and
build resources for Nuitka.

```bash
rm -rf output
mkdir -p output
cp ../CodeBase/*.py output/
cp laser.ico output/
cp -R libs output/libs
```


## Step 6: Compile with Nuitka

```bash
cd output

python -m nuitka \
    --standalone \
    --macos-create-app-bundle \
    --macos-app-icon=laser.ico \
    --macos-app-name=LaserTAG \
    --output-filename=LaserTAG \
    --enable-plugin=pyside6 \
    "--include-data-files=libs/*.dylib=libs/" \
    LaserTAG.py

cd ..
```

Verify the bundle contains the dylibs:

```bash
ls output/LaserTAG.app/Contents/MacOS/libs/libmpv.2.dylib && echo "OK" || echo "MISSING"
```

Test the binary directly:

```bash
./output/LaserTAG.app/Contents/MacOS/LaserTAG
```


## Step 7: Create DMG installer

```bash
mkdir -p output/dmg_staging
cp -R output/LaserTAG.app output/dmg_staging/
ln -s /Applications output/dmg_staging/Applications
hdiutil create \
    -volname LaserTAG \
    -srcfolder output/dmg_staging \
    -ov \
    -format UDZO \
    output/LaserTAGInstaller.dmg
rm -rf output/dmg_staging
```

Verify:

```bash
ls -lh output/LaserTAGInstaller.dmg
```


## Step 8: Install locally

```bash
cp -R output/LaserTAG.app /Applications/
```


## Step 9 (optional): Clean up staged source from output

The output directory still contains copies of the .py files and
laser.ico used during compilation. These are not needed after build.

```bash
rm -f output/*.py output/*.ico
rm -rf output/libs
```

This leaves only LaserTAG.app, LaserTAGInstaller.dmg, and the Nuitka
build/dist directories in output/.


## Troubleshooting: If MacOS blocks the app or marks it as corrupt

This app is not signed with an Apple Developer ID, so Gatekeeper may
block it on first launch. Three ways to bypass this:

Option 1 (GUI):
  Right-click LaserTAG.app, choose Open, then click Open in the
  dialog. Only required once per machine.

Option 2 (Terminal):

```bash
xattr -cr /Applications/LaserTAG.app
```

Option 3 (System Settings):
  After a blocked launch attempt, open System Settings, go to
  Privacy and Security, scroll down, and click Open Anyway next
  to the LaserTAG message.


## Troubleshooting: Rebuilding libs after Homebrew update

To update mpv or ffmpeg via Homebrew, delete libs/ and repeat
from Step 1:

```bash
rm -rf libs
brew install mpv
```

Then run Steps 1 through 4 again before rebuilding.


## Troubleshooting: Missing symbol or library errors at launch

If the app crashes with "Symbol not found" or "Library not loaded",
check which library is named in the error message.

If it is a library LaserTAG does not need (like vapoursynth), create
a stub following the pattern in Step 3.

If it is a real dependency, copy it from Homebrew, rewrite its load
path, and codesign it:

```bash
cp /opt/homebrew/lib/libWHATEVER.dylib libs/
install_name_tool -id "@loader_path/libWHATEVER.dylib" libs/libWHATEVER.dylib
codesign --force --sign - libs/libWHATEVER.dylib
```

Then rebuild from Step 5.
