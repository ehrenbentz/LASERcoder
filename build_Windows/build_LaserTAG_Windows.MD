# LaserTAG Windows Build Instructions

All commands are run from inside `LaserTAG\build_Windows\` using a
standard Windows Command Prompt (cmd.exe), not WSL or PowerShell.

```
LaserTAG\
  CodeBase\            Python source files
  build_Windows\       Build resources and scripts
    libmpv-2.dll       MPV library for Windows
    laser.ico          Application icon
    LaserTAG.iss       Inno Setup installer script
    build_LaserTAG_Windows.bat   Automated build script
    output\            Build artifacts (generated by Step 1)
      LaserTAG.build\
      LaserTAG.dist\   Complete application folder
        LaserTAG.exe
        libmpv-2.dll
        ...
      LaserTAGSetup.exe  Installer (generated by Step 5)
  build_MacOS\         macOS build resources
```


## Prerequisites

```
pip install nuitka PySide6 python-mpv
```

A C compiler is required. Nuitka will download MinGW automatically on
first run if MSVC is not installed. Alternatively, install Visual Studio
Build Tools with the "Desktop development with C++" workload.

For installer creation: Inno Setup 6 from https://jrsoftware.org/isinfo.php


## Step 1: Prepare output directory

Create a clean output directory and stage the source code and build
resources for Nuitka.

```
rmdir /s /q output
mkdir output
copy ..\CodeBase\*.py output\
copy laser.ico output\
copy libmpv-2.dll output\
```


## Step 2: Compile with Nuitka

```
cd output

python -m nuitka ^
    --standalone ^
    --windows-console-mode=disable ^
    --windows-icon-from-ico=laser.ico ^
    --output-filename=LaserTAG.exe ^
    --enable-plugin=pyside6 ^
    --include-data-files=libmpv-2.dll=. ^
    --windows-file-version=1.0.0.0 ^
    --windows-product-version=1.0.0.0 ^
    --windows-company-name="Cornell University" ^
    --windows-product-name=LaserTAG ^
    --windows-file-description="LaserTAG - Lightweight application for scoring ethology recordings and Tracking Animals Gooder" ^
    LaserTAG.py

cd ..
```

Verify the executable exists:

```
dir output\LaserTAG.dist\LaserTAG.exe
```

Test it:

```
output\LaserTAG.dist\LaserTAG.exe
```


## Step 3 (optional): Clean up staged source from output

The output directory still contains copies of the .py files and build
resources used during compilation. These are not needed after build.

```
del output\*.py
del output\laser.ico
del output\libmpv-2.dll
```

This leaves only LaserTAG.build\, LaserTAG.dist\, and (later)
LaserTAGSetup.exe in output\.


## Step 4 (optional): Code signing

If you have a code signing certificate (.pfx file), sign the executable
before creating the installer. This prevents Windows SmartScreen
warnings for end users.

If you have not purchased a valid certificate, SignTool will throw an error (below).
This is normal if you have a self-signed certificate. It does not imply that the code was not properly signed.
"SignTool Error: A certificate chain processed, but terminated in a root
        certificate which is not trusted by the trust provider."

```
signtool.exe sign /f "C:\path\to\YourCert.pfx" /p YOUR_PASSWORD /fd sha256 /td sha256 /tr http://timestamp.digicert.com /a output\LaserTAG.dist\LaserTAG.exe
```

Verify the signature:

```
signtool.exe verify /v /pa output\LaserTAG.dist\LaserTAG.exe
```


## Step 5: Create installer with Inno Setup

The `LaserTAG.iss` script is configured to read from
`output\LaserTAG.dist\` and write the installer to `output\`.

```
"C:\Program Files (x86)\Inno Setup 6\ISCC.exe" LaserTAG.iss
```

Verify:

```
dir output\LaserTAGSetup.exe
```

If the installer was also signed, you can sign it too:

```
signtool.exe sign /f "C:\path\to\YourCert.pfx" /p YOUR_PASSWORD /fd sha256 /td sha256 /tr http://timestamp.digicert.com /a output\LaserTAGSetup.exe
```


## Distribution

### Option A: Installer (recommended)

Distribute `output\LaserTAGSetup.exe`. It installs LaserTAG to
Program Files, creates Start Menu and Desktop shortcuts, and adds
the install directory to the user's PATH.

### Option B: Portable zip

```
cd output
ren LaserTAG.dist LaserTAG
powershell Compress-Archive -Path LaserTAG -DestinationPath LaserTAG.zip
cd ..
```

Distribute `output\LaserTAG.zip`. Users extract it and run
`LaserTAG.exe` directly. No installation required.


## Troubleshooting: libmpv-2.dll not found at runtime

If the compiled application fails with an error about libmpv-2.dll,
verify that LaserTAG.py includes the PATH patch for Windows:

```python
if sys.platform == "win32":
    os.environ["PATH"] = current_dir + os.pathsep + os.environ.get("PATH", "")
```

This must appear before `import mpv`.

Also verify the DLL was included in the build output:

```
dir output\LaserTAG.dist\libmpv-2.dll
```

If missing, copy it manually:

```
copy libmpv-2.dll output\LaserTAG.dist\
```


## Troubleshooting: PySide6 vs PyQt6

The codebase uses PySide6. If Nuitka warns about PyQt6, it means
PyQt6 is installed in the Python environment alongside PySide6.
This is harmless (Nuitka excludes it) but produces a warning. To
silence it:

```
pip uninstall PyQt6 PyQt6-Qt6 PyQt6-sip
```


## Troubleshooting: Nuitka cannot find a C compiler

If Nuitka fails with a compiler error, either:

1. Let Nuitka download MinGW automatically (it will prompt on first run).

2. Install Visual Studio Build Tools:
   https://visualstudio.microsoft.com/visual-cpp-build-tools/
   Select "Desktop development with C++" during installation.


## Troubleshooting: Windows SmartScreen warning

If the executable is not code-signed (Step 4 was skipped), Windows
SmartScreen will show a warning on first launch. Users can bypass it:

1. Click "More info" on the SmartScreen dialog.
2. Click "Run anyway".

This is a one-time prompt per machine. Signing the executable with a
valid certificate eliminates this warning entirely.
